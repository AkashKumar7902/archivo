Archivo Ontology Tutorial

This page is dedicated to explain how to deploy a ontology correctly so Archivo can use it (which is the recommended way of the W3C). It also explains how to easily archieve the 4 star rating for a vocabulary in Archivo.

Ontology Deployment

Generally speaking, Archivo can process the ontology if it follows the recommendations of the W3C for publishing RDF vocabularies, so if your ontology is not deployed yet you can follow the tutorial there. But there are few extra requirements/features of Archivo:

  • Ontology Recognition: To be recognized as a vocabulary, the RDF content of the ontology should contain a triple in the form of:
    <NIR> rdf:type owl:Ontology . OR <NIR> rdf:type skos:ConceptScheme .
    The NIR is the non information resource of the ontology, e.g for FOAF it is http://xmlns.com/foaf/0.1/, and acts as the representation of the ontology.

  • RDF Accessablity: The vocabulary's RDF content must be accessible from this NIR (non information resource), either directly or via content negotiation. Here Archivo not only supports RDF+XML, but also Turtle and N-Triples. If there are multiple formats available, Archivo chooses the one with the most triples.

  • Note: Archivo does not (yet) support slash ontologies that only link to its classes/properties and not define them (e.g http://dbpedia.org/ontology/)

Ontology Sources

Auto discovery: Each week DBpedia Archivo follows links from the known part of the Semantic Web to discover more ontologies. In particular, Archivo analyses: prefix.cc, http://lov.okfn.org, VoID data from Databus artifacts, outgoing links (e.g. owl:import, owl:equivalentClasses|Properties) from archived ontologies.

User Addition: If Archivo couldn't find your ontology it can be added here just by pasting the URI.

Archivo Stars

The Archivo Stars provide additional tests to check the fitness of the vocabulary for the semantic web. This section explains how to quickly fix problems with a vocabulary to earn all four Archivo Stars.

★ Retrieval & Parsing

This star is granted if the ontology is retrievable from its non information resource and parses correctly. So if the ontology is deployed following the recommendations written above, additionaly it should parse without errors. There are three different ways to debug it, depending on which step of deployment the vocabulary is:

  1. Local as a file:Archivo uses the Raptor RDF library for parsing, so to test the local ontology file use it by running
    rapper -i [file-type] [file-path]
    and it will return the parsing errors. Additionally the ontology should be deployed following W3C's recommendations.

  2. Deployed: If the ontology is accessable online paste the URI of it into the add feature and either it will add it to Archivos index or show the problem of the deployment.

  3. Already on Archivo: If the ontology is already on Archivo you can see the errors Raptor RDF yielded on the ontology view of the vocabulary in the column Retrieval & parsing.

If the ontology is deployed correctly and Raptor RDF can parse it is granted the first star.

★ License (I & II)

The second and third star are granted for using a license, the second star for using one at all and the third for using a homogenized license with the property dct:license and a URI as object. The best way here is making it correct from the start and adding a triple like 

<NIR> dct:license <my-license-uri> .
to your ontology. By adding this simple triple the vocabulary earns the second and third star at once. If you're not sure what license(s) to choose you can search one that fits your needs at dalicc.
In the near future the third star will only be granted for using license URIs that are machine readable.

★ Consistency

The forth star is granted for logical consistency of the ontology (tested with Stardog/Pellet). You can test it yourself by using the Pellet CLI or just access the consistency check of Archivo at the ontology view. There a pellet report is available at the failed consistency check symbol ✘ where the logical inconstistencies can be seen and fixed.
